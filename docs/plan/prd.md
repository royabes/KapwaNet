# KapwaNet – Product Requirements Document (PRD)

## 1. Product Name

**KapwaNet**

## 2. Vision

KapwaNet is a community platform rooted in the concept of *kapwa* (shared humanity). It enables organizations to run dignified bayanihan (mutual aid), share essential goods, and communicate internally through a branded, easy-to-deploy digital platform.

KapwaNet is designed to be:

* Peer-to-peer (platform, not a service provider)
* Organization-first (deployable per org)
* Dignity-centered (not charity-only)
* Open-source and extensible

## 3. Target Users

* Community organizations (nonprofits, cultural orgs, churches, settlement agencies)
* Members and volunteers
* Individuals requesting or offering help
* Organization admins and moderators

## 4. Core Value Proposition

* One platform for **public website + member portal**
* Built-in **bayanihan help** and **item sharing**
* Strong **styling and layout templates** (no design skills needed)
* Easy VPS or managed deployment

## 5. In-Scope MVP Features

### 5.1 CMS & Public Website

* Editable pages (Home, About, Programs, Get Involved, Contact)
* News & announcements
* Media library
* Draft/review/publish workflow

### 5.2 Styling & Templates (Key Differentiator)

* Theme tokens (colors, fonts, spacing, buttons)
* Curated theme presets
* Block-based page templates
* Template library (Home, About, Programs, Campaigns, Portal)
* Replace text/images, reorder blocks, preview before publish

### 5.3 Bayanihan Help Module

* Post help requests or offers
* Categories (non-regulated only)
* Matching and messaging
* Status flow (Open → Matched → Completed)
* Safety reporting and moderation

### 5.4 Item Sharing Module

* Offer/request items (food, clothing, essentials)
* Reservation and pickup coordination
* Food-specific fields (expiry, allergens)
* Prohibited items enforcement

### 5.5 Internal Communications

* Internal announcements
* Member discussions (basic)
* Direct messaging

### 5.6 Moderation & Trust

* Reporting (posts, users, messages)
* Moderator dashboard
* Actions: warn, remove, suspend
* Audit logs

## 6. Explicit Non-Goals (MVP)

* No payments between users
* No booking or gig marketplace
* No AI-generated content (future phase)
* No professional verification

## 7. Roles & Permissions

* Platform Admin
* Organization Admin
* Moderator
* Member

## 8. Legal & Compliance Posture

* Peer-to-peer platform only
* No endorsements or guarantees
* Alberta PIPA-aligned data handling
* Approximate location only

## 9. Success Metrics

* Active members per org
* Help requests fulfilled
* Items shared
* Time to match
* Admin satisfaction with setup

## 10. Roadmap Summary

* Phase 1: MVP (core + templates)
* Phase 2: Groups, events, volunteer board
* Phase 3: Federation, AI assist, sponsors
